# 專題實作路線圖 (Actionable Roadmap)

這個系統架構非常龐大，如果是作為「專題」來實作，絕對不能一開始就想把架構圖上所有的東西一次做完，否則非常容易遇到挫折並迷失方向。

建議採用 **敏捷式開發 (Agile) 與最小可行性產品 (MVP, Minimum Viable Product)** 的策略，將整個大專題拆解為具體的里程碑，每次只做好一件小事並驗證它。

以下為您規劃的具體實作步驟（由淺入深）：

---

## 階段一：建立開發環境與 MVP 核心 (建議週數：1~2週)
**目標**：不急著買昂貴的新硬體，先在手邊的舊筆電、舊桌機，或使用虛擬機 (如 VirtualBox/VMware) 建立開發雛型。萬丈高樓平地起！

*   **Step 1. 安裝底層系統**：在一台機器上安裝純文字介面的 **Ubuntu Server** 22.04 LTS 或 24.04 LTS。
*   **Step 2. 基礎網路設定**：設定好這台機器的靜態 IP (Static IP)，並設定好 SSH (Secure Shell) 服務。確保你能從自己平常用的個人電腦，無頭 (Headless) 遠端操控它。
*   **Step 3. 安裝容器化環境**：安裝 **Docker** 與 **Docker Compose**。這是貫穿整個專題最重要的基礎技術，未來所有服務都會跑在裡面。

## 階段二：開發「總控制介面」雛型 (建議週數：2~3週)
**目標**：做出架構圖中的「絕對軸心」，先不用做控制，先求「能讀取系統狀態」即可。

*   **Step 1. 後端 API (您架構的大腦)**：使用 Python (推薦 FastAPI 或 Flask) 或 Node.js (Express) 寫一支簡單的 Server 程式。
    *   *這週的任務*：寫一個 API 端點，透過 Python 或 Node 執行 Linux 系統指令 (如 `top`, `free -m`, `df -h`)，讀取這台機器的 CPU、RAM、硬碟使用率，並以 JSON 格式回傳。
*   **Step 2. 前端 UI (您系統的雙眼)**：使用 Vue.js、React 或純 HTML/JS 寫一個 Web 頁面。
    *   *這週的任務*：用 JavaScript 呼叫剛才寫好的後端 API，將 CPU/RAM 資料在畫面上變成動態圖表或數字。**完成這步，您就擁有專屬監控儀表板的原型了！**

## 階段三：建置第一個服務與虛擬化實作為 (建議週數：2~3週)
**目標**：開始在系統上跑實際的應用，練習「資源分配」。

*   **Step 1. NAS 與私有雲**：利用 Docker 架設一套簡單的開源私有雲（如 **Nextcloud** 或 **File Browser**）。測試是否能在區域網路內存取檔案。
*   **Step 2. 遊戲伺服器測試**：找一個您熟悉的遊戲 (如 Minecraft Server 或 Palworld)，用 Docker 跑起來，模擬架構圖中的遊戲伺服器。
*   **Step 3. 將服務整合進總控制介面**：修改階段二的後端 API，讓它能透過 Docker API 下達指令開啟/關閉指定的容器。這就是實作「自動電源管理」與「控制系統」的第一步。在儀表板上做一個按鈕，按下去可以關閉/開啟遊戲伺服器。

## 階段四：IoT 設備串接與資料庫 (建議週數：3~4週)
**目標**：打通左側的「前端 IOT 系統」與「NAS 資料庫」。

*   **Step 1. 準備 IoT 硬體**：買一片便宜的開發板 (如 ESP32、Arduino 或是 Raspberry Pi Pico W) 加上溫濕度感測器 (DHT11/DHT22)。
*   **Step 2. 架設通訊層與資料庫**：在 Ubuntu 上用 Docker 把 **Mosquitto (MQTT Broker)** 與 **InfluxDB (時序資料庫)** 架設起來。
*   **Step 3. 資料上傳雙向溝通**：寫 C++ (Arduino IDE) 讓 ESP32 讀取溫濕度，透過 WiFi 與 MQTT 協定，將資料送回您的伺服器，再由您的後端程式存入 InfluxDB。
*   **Step 4. UI 呈現 IOT 數據**：在「總控制儀表板」網頁上新增一個區塊，顯示並畫出感測器傳回的歷史歷史溫濕度折線圖。至此，您的「智慧屋」就成型了。

## 階段五：AI 客服與資安防護 (建議週數：3~4週)
**目標**：加入專題進階亮點，並完善安全防護 (符合 #資訊安全 訴求)。

*   **Step 1. 導入 AI 客服**：
    *   *輕量版*：寫後端程式串接 OpenAI API 或 Gemini API，在您的儀表板做一個聊天對話框，可以跟它對話。
    *   *進階版 (專屬 RAG 系統)*：如果硬體有獨立顯卡，使用 Docker 跑 **Ollama** 本地端模型，並寫程式讓模型可以讀取 NAS (Nextcloud) 裡的文件來回答問題。
*   **Step 2. 防火牆與 VPN**：設定 Linux 的 `UFW` 防火牆擋住所有危險 Ports。使用 Docker 建置 **WireGuard VPN** 伺服器，嘗試用手機在外面上網時，撥接 VPN 連回專題伺服器，測試是否能安全地內網穿透。

## 階段六：系統優化與專題成果收斂 (建議週數：3~4週)
**目標**：測試架構穩定性並準備結案。

*   **Step 1. 手機端 App (選修)**：如果您對手機開發有興趣，可以使用 Flutter/React Native 寫一個簡單的 App。核心概念：App 只有最低權限，只能讀取 API 資料或控制 IoT，不能去動伺服器底層開關。
*   **Step 2. 腳本整合 (省電目標)**：完善您的電源管理 Cronjob 腳本。
*   **Step 3. 專題報告與 Demo 準備**：將這套系統的架構圖重新精緻化繪製，撰寫 API 說明文件，並錄製操作展示影片。
